第一章 Agent 工具机制
1.1 什么是 Agent 工具
Claude Code 内置了一个 Agent 工具，允许主对话启动子智能体（Subagent）来处理复杂任务。子智能体拥有独立的上下文窗口，完成后将结果返回给主对话。
核心价值：
- 并行处理：一条消息可同时启动多个子智能体
- 上下文保护：子智能体的探索结果不会污染主对话的上下文
- 专业化分工：不同类型的子智能体擅长不同的任务
1.2 Agent 工具参数
Agent 工具通过以下参数进行配置：
参数	类型	必填	说明
description	string	是	3-5 个词的任务描述，用于标识 Agent
prompt	string	是	Agent 的完整任务指令（必须自包含）
subagent_type	string	否	子智能体类型：general-purpose、Explore、Plan、claude-code-guide、statusline-setup
model	string	否	模型覆盖：sonnet、opus、haiku
run_in_background	boolean	否	是否后台运行，默认 false
isolation	string	否	隔离模式："worktree" 创建独立工作区

1.3 子智能体的生命周期
创建（主对话调用 Agent 工具）
  ↓
初始化（独立上下文、加载工具）
  ↓
执行（处理 prompt、调用工具）
  ↓
返回结果（单条消息返回主对话）
  ↓
销毁（上下文释放）+


关键规则：
- 子智能体启动后是一个全新的智能体，没有之前对话的记忆
- prompt 必须自包含（self-contained），包含所有必要的上下文
- 结果以单条消息返回给主对话
