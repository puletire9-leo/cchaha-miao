## 二、已学完

### 1. Query Engine 入口
```typescript
export async function* query(params: QueryParams)
```

### 2. AsyncGenerator
- 边处理边返回，不是等全部完成
- 代价：复杂度
- 替代：回调、Promise（都有问题）

### 3. Message 格式（4种）
- user：用户输入
- assistant：AI 输出，可能带 tool_use
- tool_use：AI 想调用工具
- tool_result：工具执行结果

### 4. System Prompt
- AI 的"身份和规则"
- 结构：身份 + 工具列表 + 使用规则

### 5. 数据流
```
用户输入
     ↓
解析（命令/消息/Skill）
     ↓
createUserMessage() → Message
     ↓
+ System Prompt + 历史消息
     ↓
API 发送
     ↓
AI 处理 → Tool Use
     ↓
工具执行 → Tool Result
     ↓
循环或结束
```

### 6. while(true) 循环
- 不知道要循环多少次
- 用 break/return 结束

## 三、学习方法（思维转变）

```
1. 找入口和出口
2. 看数据流
3. 问"为什么这样设计"
4. 理解代价和选择
```

核心问题：
- "为什么要这样设计？"
- "不这样会怎样？"
- "设计者担心什么？"
